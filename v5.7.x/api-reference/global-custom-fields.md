# Global Custom Fields

The Global Custom Fields API allows you to create and manage custom fields that can be used across all subscriber lists within the Octeth platform, providing consistent data collection and organization.

::: warning Note
This section is under development. Documentation will be available soon.
:::

## API Endpoints Covered

This section will document the following global custom field management endpoints:

- **Custom Field Management**
  - `Global.CustomField.Create` - Create a new global custom field
  - `Global.CustomField.Update` - Update existing custom field properties
  - `Global.CustomField.Delete` - Remove a global custom field
  - `Global.CustomField.Get` - Retrieve specific custom field details

- **Field Configuration**
  - `Global.CustomFields.Get` - List all global custom fields
  - `Global.CustomField.SetOptions` - Configure field options and values
  - `Global.CustomField.GetOptions` - Retrieve field option configurations
  - `Global.CustomField.Validate` - Validate custom field data

- **Field Types Support**
  - Text and textarea fields
  - Number and date fields
  - Dropdown and multi-select options
  - Checkbox and radio button fields
  - URL and email validation fields
  - Custom validation patterns

- **Integration Features**
  - Cross-list field synchronization
  - Default value management
  - Required field enforcement
  - Field visibility and permissions
  - Import/export field mapping
