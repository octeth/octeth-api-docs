# Delivery Servers

The Delivery Servers API provides comprehensive management of email delivery infrastructure, allowing you to configure, monitor, and optimize SMTP servers and email sending services.

::: warning Note
This section is under development. Documentation will be available soon.
:::

## API Endpoints Covered

This section will document the following delivery server management endpoints:

- **Server Management**
  - `DeliveryServer.Create` - Add a new delivery server
  - `DeliveryServer.Update` - Update server configuration
  - `DeliveryServer.Delete` - Remove a delivery server
  - `DeliveryServer.Get` - Retrieve specific server details

- **Server Listing**
  - `DeliveryServers.Get` - List all configured delivery servers
  - `DeliveryServers.GetActive` - Retrieve active servers only
  - `DeliveryServers.GetStats` - Get server performance statistics
  - `DeliveryServers.GetUsage` - Monitor server usage and quotas

- **Server Configuration**
  - SMTP server settings and authentication
  - API-based delivery service integration
  - Server rotation and load balancing
  - Rate limiting and throttling controls
  - Bounce and complaint handling
  - Server priority and weight settings

- **Monitoring & Analytics**
  - Real-time delivery monitoring
  - Server health and status tracking
  - Performance metrics and analytics
  - Error logging and troubleshooting
  - Delivery success rates
  - Server reputation monitoring
  - Usage quotas and limits tracking
